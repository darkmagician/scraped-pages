# 套餐概览

2026-04-30 05:55:19

**Coding Plan** 是一款为开发者量身定制的 AI 编程订阅服务，旨在全方位助力高效编码与项目开发。

1. **模型自由切换**：一次订阅即可在多种主流 Code 模型间按需切换，包括：`DeepSeek-V3.2`、`GLM-5`、`GLM-4.7`、`MiniMax-M2.5`、`Kimi-K2.5`、`Kimi-K2-Turbo`、`Qwen3-Coder`。

2. **兼容主流工具**：完美适配多种开发场景，支持 Claude Code、OpenCode、OpenClaw、Roo Code、Cursor等主流 AI 编码工具，多工具之间套餐额度共享。

3. **灵活套餐选择**：提供 Lite 与 Pro 两款套餐，精准匹配普通日常开发与高强度编程场景。

4. **平稳极速体验**：调用高峰期无明显降速，TPM（每分钟 Token 数量）充足，保障正常开发需求不中断。


## 适用场景及要求

1. **个人开发场景**：专为个人开发者打造，助力完成个人项目开发、新技术学习实践、脚手架与工具搭建等编码任务。

2. **企业级提示**：如果您有企业级的开发需求，请勿使用此个人订阅，请通过 [JoyBuilder](https://joybuilder-console.jdcloud.com/smart-square/model/list) 获取专业的 API 模型调用服务。


## 核心使用规范

1. **生效范围**：套餐额度 **仅在指定的 AI 编程工具中生效**，多工具共享额度。

2. **禁止 API 滥用**： **严禁用于常规 API 调用**。如果在非 AI 编程工具中使用 Coding Plan 专属的 `Base URL` 和 `API Key`，系统极可能将其识别为滥用或违规，这 **会导致您的订阅被停用或账号被封禁**。


## 套餐详情

套餐用量说明：

| **套餐** | **适用人群** | **用量限制** |
| --- | --- | --- |
| Lite套餐 | 中等强度的开发者，适合大多数开发者。 | 1. 每5小时：最多约 1,200 次请求。<br>2. 每周：最多约 9,000 次请求。<br>3. 每订阅月：最多约 18,000 次请求。 |
| Pro套餐 | 复杂项目开发，适合高强度工作的开发者。 | Lite套餐的5倍用量。<br>1. 每5小时：最多约 6,000 次请求。<br>2. 每周：最多约 45,000 次请求。<br>3. 每订阅月：最多约 90,000 次请求。 |

表格中的“请求次数”为模型调用的 **预估值**。在 AI 编程工具中， **用户的一次提问通常会触发多次模型背后的自动调用**，每次调用均会计入额度消耗。

- **简单问答/代码生成**：单次提问通常触发 **5-15 次** 模型调用。

- **代码重构/复杂任务**：单次提问通常触发 **15-30 次或更多** 的模型调用。

- _注：实际可用量会因项目复杂度、代码库规模、是否开启“深度思考 等因素而存在差异。您可以随时在开通管理页面查看实时用量。_


## 套餐有效期

套餐有效期以“自然月”为单位，自购买成功当日起算。示例：

- 01月04日购买 1个月 -> 02月04日 23:59 到期

- 01月31日购买 1个月 -> 02月28日 23:59 到期（遇平月月底）

- 02月01日购买 1个月 -> 03月01日 23:59 到期


## 套餐额度刷新规则

套餐时间周期内额度耗尽后，需等待下个周期自动恢复， **该过程不会扣除您的其他资源包或账户余额**。

- **每 5 小时限额**：滑动窗口机制，根据实际请求发生的时间动态刷新。

- **周限额**：每周一 00:00:00 自动重置。

- **月限额**：每订阅月的第 1 日 00:00:00 自动重置。


## 订阅管理

- **退订政策**：Coding Plan **不支持退订**，购买前请根据自身需求谨慎选择。

- **升级套餐**：支持将 Lite 套餐升级至 Pro 套餐。升级费用按剩余天数折算差价，升级后，当月剩余时间的可用额度将在原有剩余额度的基础上按比例增加。

- **降级套餐**：不支持从 Pro 套餐直接降级至 Lite 套餐。如需降级，需等待当前 Pro 套餐到期后，重新购买 Lite 套餐。


## 接入编程工具

为确保正常消耗 Coding Plan 额度，配置时请务必注意以下事项：

1. **专属凭证**：必须使用 Coding Plan 专属支持的模型名称、指定的 `Base URL` 和 `API Key`。

2. **模型限制**：不支持的模型无法在 Coding Plan 中使用。

3. **配置错误**：若未使用指定的 `Base URL` 或 `API Key`，将无法使用套餐额度。若误用了通用的 API 调用 URL 和 API Key，将无法消耗 Coding Plan 额度，且 **会产生常规 API 请求的按量扣费**。
